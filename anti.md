# 🚀 Antigravity 마스터 가이드 (초심자용)

Antigravity는 구글 딥마인드에서 개발한 차세대 **"에이전틱 코딩 어시스턴트(Agentic Coding Assistant)"**입니다.
단순히 코드 몇 줄을 짜주는 채팅 봇(gemini-cli나 구형 챗봇)이 아니라, 기획, 쉘 명령어 실행, 외부 API(MCP) 통신, 지식 축적까지 자율적으로 수행하는 **"또 하나의 개발자(Agent)"**입니다.

이 가이드를 통해 Antigravity 시스템의 3대 핵심 구조를 이해하고 완벽하게 통제하는 방법을 배울 수 있습니다.

---

## 💡 1. 핵심 개념: Antigravity는 어떻게 생각하는가?

기존 AI 봇들은 매번 프롬프트에 "이 프로젝트는 리액트고, Tailwind를 쓰며..." 라고 길게 주입해야 했지만, Antigravity는 **스스로 환경을 스캔하고 규칙을 찾아 읽습니다.**

- **자율성(Agency)**: 사용자가 "저 폴더에 있는 파일들 체크해 줘"라고만 해도, 스스로 터미널에서 `ls`, `grep` 등의 명령어를 실행해 파일을 찾고 복합적인 문제를 순차적으로 해결합니다.
- **루트(Root) 중심주의**: 시스템 오작동(다른 프로젝트의 규칙 충돌)을 막기 위해, 에디터를 처음 연 **최상위 폴더(Workspace Root)의 설정만 인식**합니다. 하위 디렉토리에 몰래 숨겨둔 룰이나 스킬은 무시하므로 모든 통제는 최상위에서 이루어져야 합니다.

---

## 🛠️ 2. 관제탑 구축: `.agent/` 폴더 활용법

Antigravity를 내 입맛대로 조종하는 핵심 컨트롤 타워는 프로젝트 루트에 위치한 **`.agent/`** 디렉토리입니다. 이 안에 파일들을 배치하여 AI의 지능과 행동 반경을 확장할 수 있습니다.

### ① 공식 지원 (시스템 하드코딩 - 강제 발동)

시스템이 명시적으로 인식하여 무조건 우선순위로 발동하는 기능들입니다.

- **`workflows/` (슬래시 명령어)**
  - **용도**: 매번 길게 치기 귀찮은 반복 작업(예: 디자인 추출 후 검증, 배포 파이프라인)을 자동화합니다.
  - **작동 방식**: `.agent/workflows/figma-svg.md` 라는 파일을 생성해 두면, 채팅창에 `/figma-svg` 라고 치는 순간 파일에 적힌 1~5단계의 스크립트와 체크리스트가 강제 실행됩니다.
  - **Tip**: 파일 안에 `// turbo`나 `// turbo-all` 주석을 달면 쉘 명령어 실행 시 사용자 승인 과정을 건너뛰고 백그라운드에서 광속으로 처리됩니다.

- **`skills/` (스킬 트리거)**
  - **용도**: 특정 작업을 수행할 때 AI가 반드시 지켜야 할 스크립트 모음과 체크리스트입니다. (`.claude/hooks` 역할 대체)
  - **작동 방식**: `.agent/skills/figma_processor/SKILL.md` 처럼 생성합니다. 파일 상단 설명(Description)을 읽고, AI가 "아, 이 유저가 Figma 작업을 시켰으니 이 스킬북을 먼저 읽고 시작해야겠다"라고 **스스로 판단하여 지침을 꺼내 읽습니다.**

### ② 컨벤션 (권장 패턴 - AI 지식 베이스)

강제 발동은 아니지만, AI가 작업 전/중간에 컨텍스트(문맥)를 파악하기 위해 **자율적으로 샅샅이 뒤져서 참고하는 참고서**들입니다.

- **`rules/` (프로젝트 전역 규칙)**
  - **예시**: `project-guidelines.md`
  - 프로젝트의 구조(디렉토리 미러링 등), 기술 스택, 디자인 토큰, 네이밍 컨벤션을 적어둡니다. (기존 `.cursorrules`나 `CLAUDE.md` 의 완벽한 대체재)
- **`examples/` (모범 사례)**
  - AI에게 "코드 짤 때 모르겠으면 이 폴더에 있는 `good-api-pattern.ts` 를 보고 똑같이 짜" 라고 유도하여 결과물의 퀄리티 기복을 없앱니다.
- **`templates/` 및 `prompts/`**
  - 반복되는 보일러플레이트 구조나 긴 프롬프트를 저장해 두고 단축어처럼 활용합니다.

## 🧠 3. 기억의 구조: 메모리와 지식 시스템

프로젝트(`.agent/`)를 벗어나서, Antigravity 시스템 자체가 사용자(YOU)와 상호작용하며 쌓는 두 가지 거대한 기억 장치가 있습니다.

### ① User Global Memory (전역 헌법)

어떤 프로젝트(폴더)를 열든 **무조건 예외 없이 최우선으로 지켜야 하는 "사용자와 AI 간의 절대 규칙(에티켓)"** 입니다.

- **무엇을 넣어야 할까? (Best)**
  - "모든 주석은 무조건 한국어로 작성할 것"
  - "명시적 지시가 없으면 쉘에서 `rm` 이나 패키지 삭제 명령어를 마음대로 쓰지 말 것"
  - "새 프로젝트 작업 시작 전엔 가장 먼저 `.agent/rules/` 디렉토리를 샅샅이 읽어볼 것"
- **무엇을 넣으면 안 될까? (Worst)**
  - "디렉토리 구조는 무조건 `pages/`에 맞춰라." (다른 React, Vue 프로젝트를 열었을 때 심각한 오작동과 환각 유발)

### ② KIs (Knowledge Items 시스템 - 자동 축적 지식)

사용자가 직접 타이핑하지 않아도, 백그라운드의 보조 에이전트(Knowledge Subagent)가 대화 내용을 분석해 **가치 있는 해결책을 평문(Plain Text)으로 로컬 스토리지에 요약 저장**합니다.

- **어디에 저장되나요?** `~/.gemini/antigravity/knowledge/`
- **어떻게 쓰이나요?** 며칠 전 "A 프로젝트"에서 발생했던 Next.js 오류 해결법을, 오늘 "B 프로젝트"를 할 때 AI가 KIs 폴더에서 스스로 검색하여 가져와 똑똑하게 해결해 줍니다.
- **특징**: 사용자는 해당 경로의 `*.md` 파일을 열어 직접 읽고 편집하며 AI의 지식을 직접 관리하고 교정할 수 있습니다.

## 🎯 완벽한 세팅을 위한 3줄 요약 가이드

1. **글로벌 메모리**: "나라는 개발자와 일할 때 지켜야 할 선" (프로젝트 불문, 한국어 주석, 행동 제약 등)
2. **`.agent/rules/`**: "이 프로젝트에만 국한된 개발 컨벤션" (기존 CLAUDE.md 대체용)
3. **`.agent/workflows/`**: "매번 말하기 귀찮은 동작들의 자동화 단축어" (Figma 추출 시 무조건 검증하기 등)

> 이 세 가지만 명확히 분리해서 사용하신다면, Antigravity는 현존하는 그 어떤 코딩 어시스턴트보다 사용자님의 의도를 정확히 꿰뚫는 강력한 파트너가 될 것입니다!
